# 数字工作台导览

工作台是 anyCode 的**本地网页看板**：帮你看清「哪些项目在跑、结果是否可信、定时任务有没有失败」，不用记命令行。

## 你能做什么

- 浏览所有已登记的项目和会话
- 查看每次运行的状态、门禁与产出文件
- 创建定时任务、看重试与失败记录
- 生成项目/会话报告（Markdown 或 HTML）

## 怎么打开

**浏览器：**

```bash
anycode dashboard --open
```

默认地址：`http://127.0.0.1:43180`

**macOS 桌面应用：** 从 [Releases](https://github.com/qingjiuzys/anycode/releases) 安装对应平台的安装包（macOS `.dmg`、Windows `.msi`/`.exe`、Linux `.deb`/`.AppImage`），打开后会自动启动工作台。

## 对话与多模态

- **网页对话**：在项目会话中可直接粘贴或选择图片发送（需 Vision 模型）。
- **渠道**：微信、Telegram、Discord 支持图片；Telegram/Discord 语音消息在配置了 `models.speech.stt` 时会自动转写。
- 详见仓库内 [运行流说明](https://github.com/qingjiuzys/anycode/blob/main/docs/run-flow.md)。

## 项目知识库

在项目详情页可配置 **知识库路径** 并执行 **重建索引**。Desktop 发布版内置向量检索；纯 CLI 开发构建需 `knowledge-embeddings` feature。Doctor 检查会提示索引与向量状态。

## MCP 与浏览器

- **设置 → MCP 服务器**：图形化编辑 `~/.anycode/config.json` 中的 `mcp.servers`。
- **设置 → 通知 → Browser 连接器**：Desktop 版可一键启用 bundled Playwright MCP；首页快捷区也会提示启用。

## 侧栏页面一览

| 页面 | 你会看到什么 | 常见操作 |
|------|----------------|----------|
| **总览** | 项目数量、运行中会话、最近动态 | 快速判断今天有没有异常 |
| **项目** | 各工作区列表、信任度、最近活动 | 进入某个项目看详情 |
| **会话** | 按项目浏览对话与任务记录 | 点开一条会话看时间线 |
| **自动化** | 定时任务、运行记录、项目护栏 | 新建定时任务、失败时重试 |
| **资产** | 助手改过的文件列表 | 确认产出、筛选未验证 |
| **报告** | 项目/会话报告 | 生成并导出给同事 |
| **审计** | 配置类操作记录 | 追溯谁改了策略 |
| **Agent / Skills** | 角色与本地技能包 | 查看已安装技能 |
| **设置** | 登录、模型、通知、运维偏好 | 改端口、报告格式等 |

## 自动化页（定时任务）

自动化页专注**定时跑任务**，不再和 Workflow 会话混在一起：

1. **顶部概览** — 任务数、最近失败、启用中的护栏
2. **新建定时任务** — 用自然语言写「每天 8 点」，再写要让助手做什么
3. **项目护栏** — 例如：门禁失败时阻断、完成后自动生成报告
4. **任务列表 / 运行记录** — 查看每次触发是否成功，失败可点「立即重试」

需要长期后台执行时，请在本机保持调度器或桌面应用在运行。

## 语言与主题

右上角可切换 **中文 / English** 和浅色/深色主题。侧栏底部的 **文档**、**帮助** 会按当前语言打开对应说明页。

## 出问题了怎么办

| 现象 | 建议 |
|------|------|
| 页面打不开 | 先执行 `anycode dashboard`，看终端是否提示端口占用 |
| 列表一直是空的 | 先在终端对某个目录跑一次任务，再回到工作台 |
| 定时任务不触发 | 确认调度器在运行；在「运行记录」里看最近是否有失败 |
| 登录页进不去 | 本机 `127.0.0.1` 一般自动以本地用户进入 |

更多排错见 [常见问题](./troubleshooting)。
